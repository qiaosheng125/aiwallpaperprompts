## 关键经验

1. 图片站不能做成工具站样子。

图片站第一屏要让用户先看到图片质量，再决定是否点进详情页。说明文字、功能解释、SOP 不应该抢首屏。

2. 网站架构要服务日更。

第六站不是一次性项目，核心资产是“图片 + prompt + 标签 + 详情页”。因此必须有增量上传、已上传记录、缩略图、归档、R2 存储和 sitemap 扩展能力。

3. 真实图片尺寸优先于 prompt 文本判断比例。

部分 prompt 文本里会同时出现 `16:9` 和 `9:16`，脚本不能只靠文本判断比例。当前规则优先读取真实图片尺寸。

4. 统计代码给出后，默认搜索提交已完成。

以后用户给出 GA4 和 Clarity 两个代码，表示 GSC、Bing、sitemap、首页请求编入索引都已经完成。Codex 负责写环境变量、部署、验证和记录。

5. 广告暂不提前接入。

当前阶段先做收录、内容质量和用户行为数据。未来流量起来后再考虑 AdSense，不提前放空广告位。
